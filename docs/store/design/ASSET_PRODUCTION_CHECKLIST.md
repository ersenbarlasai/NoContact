# Asset Production Checklist - Nosto

## 1. Asset Sizes & Export Requirements

### iOS App Store
- [ ] **Icon:** 1024x1024 px, PNG (No alpha/transparency, no rounded corners in the asset).
- [ ] **6.7" Screenshots (x7):** 1290x2796 px, PNG or JPEG.
- [ ] **6.5" Screenshots (x7):** 1242x2688 px, PNG or JPEG.
- [ ] **5.5" Screenshots (x7):** 1242x2208 px, PNG or JPEG.

### Google Play Store
- [ ] **Icon:** 512x512 px, PNG (32-bit, with alpha channel allowed, max 1MB).
- [ ] **Feature Graphic:** 1024x500 px, PNG or JPEG. Needs to be simple, with central branding. Avoid placing text near the edges.
- [ ] **Phone Screenshots (x7):** 1080x1920 px (or similar 16:9 ratio), PNG or JPEG.

## 2. Naming Conventions
To ensure smooth upload and organization, name exported files systematically:
- `iOS_67_01_Home.png`
- `iOS_67_02_SOS.png`
- `Android_01_Home.png`
- `Icon_1024.png`
- `PlayStore_Feature_1024x500.png`

## 3. Light / Dark Background Considerations
- Because the screenshots have soft Cream/Sage backgrounds, verify that they do not blend entirely into the App Store's white background (Light Mode). 
- **Action:** Add a very subtle, soft drop shadow or a faint 1px border (`#E5E5E5`) to the edge of the screenshot canvas if needed for definition.

## 4. Visual Review Checklist
- [ ] Are all headlines using the exact text from `SCREENSHOT_COPY_FINAL_TR.md`?
- [ ] Is the typography consistent across all 7 screens?
- [ ] Is the device frame aligned exactly the same on every screen? (Prevents jumping when the user swipes).
- [ ] Is the mock data inside the device screens legible?
- [ ] Does the UI strictly avoid any dating, medical, or "ex back" visual clichés?
- [ ] Is the privacy/lock badge clearly visible on Screen 7?

## 5. Store Compliance Notes
- Do not include the word "Free" or price information in the screenshots.
- Do not include Android hardware in iOS screenshots, and vice versa.
- Ensure no real user data is visible.
