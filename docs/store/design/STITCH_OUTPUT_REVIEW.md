# Stitch Output Review - Still / NoContact

This document reviews the initial visual assets generated via Google Stitch and provides guidelines for their final production and implementation.

## 1. App Icon Selection
**Selected Concept:** `AppPics/icons/pause_reflect_icon`
- **Why it works:** The soft circular ring with a minimalist "pause" symbol perfectly encapsulates the "Still" philosophy. It implies taking a moment to breathe before reacting. It uses a calming Sage background without being medical or romantic.
- **Constraints for Final Render:**
  - Must use exact brand colors: Sage background, Cream or Terracotta for the symbol.
  - No text or "AI" labels.
  - No broken hearts or medical crosses.
  - Ensure the pause bars and ring have enough thickness to be legible at small sizes (e.g., iOS Settings icon size).

**Rejected Concepts:**
- Any concepts featuring hearts, bandages, or chat bubbles were rejected as they trigger anxiety or evoke dating/medical app clichés.
- High-contrast, neon, or complex gradients were rejected for lacking the required calmness.

## 3. Required Fixes Before Store Production
To convert these references into final store assets, the following adjustments MUST be made:
- **Phone Mockups:** Ensure the device frame is not awkwardly cropped. The full top (notch/dynamic island) and bottom (home indicator area) must be visible if the device is not intentionally bleeding off the bottom edge.
- **Backgrounds:** Maintain strict use of Cream and Sage.
- **Safe Data:** No real user data. Only use the approved privacy-safe mock text.
- **Claim Compliance:** Absolutely no text implying "guaranteed healing," "therapy," or emergency medical intervention.

## 5. Flutter Implementation Guidance (Important)
- **Do NOT copy HTML/CSS:** The code generated by Stitch for these mockups is for visual reference only.
- **Use Flutter Components:** When implementing UI improvements inspired by these mockups, developers MUST use the existing Flutter `Still` design tokens and widget components (e.g., `StillCard`, `AppColors`).
- **Preserve Architecture:** Do not alter Riverpod state management or navigation behavior to match a static mockup.
